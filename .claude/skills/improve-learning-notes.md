# 学习笔记改善技能 (Improve Learning Notes)

## 描述

阅读项目中的所有 Markdown 文档，按照"高三学生能读懂"的标准进行严谨、通俗的文档书写和知识总结。

## 适用场景

任何理工科学习笔记仓库（数学、物理、电子、计算机等），特别是：
- 从零开始构建知识体系的教程
- 需要对已有笔记进行质量提升
- 从 PDF 教材提炼知识点并写成可读文档

---

## 核心原则

### 原则 1：受众定位 —— 假设读者是高三学生

**知识基线**：
- 数学：初中代数 + 高中函数/三角/向量/简单微积分概念
- 物理：高中力学、电磁学基础
- 其他学科：零基础假设

**红线**：
- ❌ 不假定读者学过大学数学（线性代数、微积分要重新推导）
- ❌ 不使用"显然"、"易知"、"同理可得"等跳步词
- ❌ 不直接引用其他学科的专业术语而不解释

---

### 原则 2：动机驱动 —— 每个概念必须回答"为什么"

对于每一个引入的新概念，必须回答三个问题：
1. **是什么**（定义，用一句话）
2. **为什么需要它**（它解决了什么实际问题？没有它会怎样？）
3. **怎么用**（它在这个学科中扮演什么角色）

**写作格式**：
```
### X.X 概念名称

> **为什么要学这个？** 
> [用日常生活类比或实际问题说明这个概念的必要性。例如："如果没有XX，我们就无法解决YY问题。"]

**定义**：[一句话精确定义]

**通俗理解**：[用类比或简单例子解释]
```

---

### 原则 3：跨学科概念 —— 完整引入 + 完备推导

当学科 A 的文档需要用到学科 B 的概念时：
1. **不要只说"参考XX学科"**——读者可能没学过
2. **给出完整的定义和推导**——从学科 B 的最基础部分开始
3. **控制深度**——只讲到够用为止，不无限延伸

**格式**：
```
### X.X 预备知识：XXX（来自[学科名称]）

> **为什么要在这里讲这个？** 
> 因为后面理解[核心概念]时需要用到XXX。即使你没学过[学科名称]，读完这一节也足够用了。

[从最基础的定义开始，一步步推导到所需结论]
```

---

### 原则 4：数学推导 —— 零跳步

任何数学推导必须：
1. **明确前提假设**——列出推导中用到的所有前提条件
2. **写出每一步**——哪怕是最简单的代数变形也要写出来
3. **标注每一步的数学依据**——用了什么公式、什么定理
4. **推导完成后验证**——用特殊值代入验证结论的合理性

**格式**：
```
**前提假设**：
- 假设 1：...
- 假设 2：...

**推导过程**：

第 1 步：[操作] → [依据：XXX]

第 2 步：[操作] → [依据：XXX]

...

第 n 步：得到结论 [公式]

**验证**：当 [特殊值] 时，代入得 [结果]，与直观理解 [一致/不一致，因为...]
```

**禁止的写法**：
- "显然"
- "易知"
- "同理可得"
- "经过简单推导"
- "可以证明"（除非给出完整证明）
- "读者不难验证"

---

### 原则 5：图例 —— 必须用 Python 绘制

所有图形、图表、波形图使用 Python（matplotlib + numpy）绘制：

1. **在文档中嵌入 Python 代码块**（完整可运行）
2. **说明图表的读法**——标注关键区域、坐标含义、不同颜色代表什么
3. **图表必须自包含**——不依赖外部理解

**格式**：
````
```python
import numpy as np
import matplotlib.pyplot as plt

# [注释说明每一步在做什么]
...

plt.savefig('images/xxx.png', dpi=150, bbox_inches='tight')
```

![图表说明](images/xxx.png)

> **图表解读**：
> - **横轴**：[含义]
> - **纵轴**：[含义]  
> - **红色区域**：[含义]
> - **关键点**：[标注的关键位置的含义]
````

---

### 原则 6：文档结构规范

每份教程文档应包含：
1. **引言**：本文档要讲什么、需要什么预备知识（诚实列出）
2. **目录**：带锚点的层级目录
3. **正文**：按"是什么→为什么→怎么用"组织
4. **小结**：本节核心结论的列表
5. **公式速查表**：所有关键公式汇总
6. **术语对照表**：中英文对照

---

## 执行流程

当此技能被调用时，按以下步骤执行：

### 第一步：全面阅读
- 用 Glob 列出所有 `*.md` 文件
- 阅读每一个文件，建立全局知识图谱（哪些概念之间有依赖关系？）

### 第二步：质量评估
对每个文档评估：
- 是否解释了每个概念的动机？
- 是否有跳步的数学推导？
- 是否缺少必要的图例？
- 是否引用了未解释的外部概念？
- 文档结构是否完整（引言/目录/正文/小结/公式表/术语表）？

### 第三步：制定改善计划
- 列出需要添加/修改的内容
- 按优先级排序：缺失的前置知识 > 跳步推导 > 缺失图例 > 结构优化

### 第四步：逐步执行
- 一次处理一个主题，确保质量
- 数学推导从最基础假设开始，不跳步
- 图例用 Python 生成，嵌入文档

### 第五步：交叉验证
- 检查新写的内容是否引用了未解释的概念
- 用特殊值代入验证数学结论

---

## 禁止事项

- ❌ 不要写"显然"、"易证"、"同理"——每一步都写出来
- ❌ 不要假设读者知道大学数学符号（如 ∫ 要解释、Σ 要解释）
- ❌ 不要使用未定义的专业术语
- ❌ 不要跳过代数变形的中间步骤
- ❌ 不要使用外部图片——所有图用 Python 生成
